# Приложение контроля склада

Многопользовательское веб-приложение для учёта материалов на складе: приход/выдача по QR, выдача пользователям, возвраты, отчёт по выработке за период.

## Возможности

1. **Пользователи и доступ**
   - Вход по логину и паролю.
   - Роль **администратор**: создание пользователей, назначение паролей и прав.
   - У каждого пользователя настраиваются права: Склад, Выдача, Выработка, Управление пользователями.

2. **Вкладка «Склад»**
   - Список всех материалов (код, название, единица, количество).
   - Добавление материала (код используется для QR).
   - Приход количества по кнопке «Приход».
   - Выдача по QR: сканирование камерой (нужен **HTTPS** на телефоне), выбор получателя и количества.

3. **Вкладка «Выдача»**
   - Выдача материала любому пользователю (выбор материала, получателя, количества).
   - Выдача и возврат по QR (сканирование материала, выбор выдачи для возврата).
   - Возврат на склад: частичный или полный (из таблицы выдач — кнопка «Возврат»).

4. **Вкладка «Выработка»**
   - Отчёт: сколько материалов взял каждый пользователь за выбранный период.
   - Учитываются выданное количество и возвраты (выработка = выдано − возвращено).
   - Выбор периода (дата с — дата по).

5. **Оформление**
   - Тёмная тема, современный интерфейс (Tailwind CSS).

## Требования

- Node.js 18+
- PostgreSQL (локально или через Docker — см. ниже)
- Docker Desktop (или Docker Engine + Compose), если поднимаете БД контейнером

## Установка

```bash
# Корень проекта
cp .env.example .env
# Отредактируйте .env: DATABASE_URL, SESSION_SECRET
# Если POSTGRES_* заданы — они должны совпадать с логином/паролем/именем БД в DATABASE_URL

npm install
npm run db:up   # PostgreSQL в Docker (пропустите, если БД уже запущена у вас локально)
npm run db:migrate
npm run db:seed

# Клиент
cd client && npm install && cd ..
```

## Запуск

Сначала убедитесь, что PostgreSQL доступен по `DATABASE_URL` (например, `npm run db:up` для контейнера из `docker-compose.yml`).

**Разработка (API + клиент отдельно):**

```bash
# Терминал 1 — сервер (порт 3000)
npm run dev

# Терминал 2 — фронтенд (порт 5173, прокси на API)
cd client && npm run dev
```

Откройте http://localhost:5173. Для входа по умолчанию: **admin** / **admin**.

**Продакшен (один сервер, раздача статики):**

```bash
npm run prod
```

Сервер слушает порт из `PORT` (по умолчанию 3000). Откройте http://localhost:3000.

## Камера и QR на телефоне

Доступ к камере в браузере на мобильном устройстве разрешён только по **HTTPS** (и по localhost при разработке). Для работы сканера QR на телефоне разверните приложение за HTTPS (например, nginx с SSL или облачный хостинг с сертификатом).

## Переменные окружения

| Переменная        | Описание                    |
|-------------------|-----------------------------|
| `DATABASE_URL`    | URL подключения PostgreSQL  |
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` / `POSTGRES_PORT` | Для `docker compose`: должны совпадать с учётными данными в `DATABASE_URL` |
| `SESSION_SECRET`  | Секрет для сессий           |
| `PORT`            | Порт сервера (по умолчанию 3000) |
| `ADMIN_LOGIN`     | Логин первого админа (seed) |
| `ADMIN_PASSWORD`  | Пароль первого админа (seed)|
